# Projects and Assignments

Estado: `planned`

## Propósito

Los proyectos integran varias skills en una tarea mayor. Deben parecer trabajo real, no un ejercicio largo disfrazado.

## Tipos

- mini proyecto de un archivo;
- proyecto interactivo;
- proyecto por milestones;
- proyecto de múltiples archivos;
- proyecto acumulativo de módulo;
- proyecto final de ruta;
- asignación importada por profesor o usuario.

## Estructura

Cada proyecto debe declarar:

- objetivo;
- contexto;
- entregables;
- archivos esperados;
- milestones;
- restricciones;
- criterios de aceptación;
- tests automáticos;
- comprobaciones manuales cuando sean inevitables;
- skills evaluadas;
- política de hints;
- política de reintentos.

## Milestones

Un proyecto puede dividirse en etapas. Cada etapa debe producir progreso real y verificable. Completar un milestone no debe bloquear la edición posterior del mismo código.

## Evaluación

La evaluación puede combinar:

- compilación;
- tests de entrada/salida;
- tests de funciones;
- escenarios interactivos;
- estructura de archivos;
- requisitos funcionales;
- revisión manual opcional.

## Reglas

- Lumen nunca sobrescribe trabajo del usuario.
- Los proyectos pueden continuar offline después de ser importados.
- Un proyecto oficial debe estar versionado.
- Los cambios de versión no reemplazan automáticamente una entrega en progreso.
- Route Mode puede usar proyectos como gates, pero debe explicar exactamente qué falta.

## Ejemplos iniciales

Blackjack, Bingo, Memory Game, sistemas de menús, inventarios con structs, persistencia en archivos y un proyecto final integrador.
